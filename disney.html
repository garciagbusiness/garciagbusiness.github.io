The Problem

Why This Was Built

The Disney+ LPO-SEO team tracks performance across 20+ global landing pages every week — from the US homepage to regional MLPs across LATAM, APAC, EMEA, and Canada. Each page generates a Looker data export (ZIP with CSVs) covering sessions, conversion rates, purchase success, daily breakdowns, plan take-rates, and category/channel attribution.

Before this pipeline, the weekly report was a 60+ page Google Doc, rebuilt from scratch by hand every week: screenshots pasted in one by one, tables retyped, charts recreated for each of the 20+ landing pages across 6 regions. A team member would spend 8–10 hours per week on this alone, and by the time it was finished, the "weekly report" was already a day stale.

The goal was to fully automate that process — replacing the manual Google Doc with an AI-driven pipeline that goes from raw Looker ZIPs to a polished, self-contained HTML dashboard delivered to stakeholders the same day, with zero manual data entry and automated English-language insights.

Architecture

How the Pipeline Works

Seven automated steps from raw Looker exports to a ready-to-share executive dashboard.

01
Drop Weekly ZIPs
Looker exports one ZIP per landing page per week. Files are dropped into a dated folder — no renaming, no reformatting required. The pipeline handles all filename matching via keyword rules.
02
DataProcessor Ingestion
Python DataProcessor class extracts each ZIP, reads 7 CSVs per page (sessions, CVR, daily breakdown, take rates, category/channel, country data), and normalizes all values with fuzzy column mapping.
03
AnalyticsEngine Computation
Per landing page: computes 4 KPIs with WoW % and direction, builds daily time series, aggregates take-rate mix, flattens category × channel conversion table, and detects anomalies via z-score thresholds.
04
Insight Generation
Rule-based engine generates plain-English summaries for each page — classifying insights as wins, warnings, or alerts based on WoW direction and anomaly flags. LLM layer adds narrative depth for executive summaries.
05
Jinja2 HTML Render
dashboard.html Jinja2 template renders the full dashboard — tabs, KPI cards, Chart.js canvas elements with inline data attributes, insight boxes, and tables — into a single self-contained HTML file.
06
Injection Scripts
Three post-render scripts enrich the HTML: inject_delta_purchases.py adds Δ column to every table; inject_first_streams.py injects Top-10 content tables from Excel; inject_plans_mlp.py adds the Plans MLP section.
Dashboard

What the Dashboard Shows

Every section of the weekly report, explained — see the live demo for the full interactive experience.

6-Tab Navigation

Sticky tab bar across Global, United States, LATAM, APAC, EMEA, and Canada. Each tab holds multiple landing page cards for that region.

KPI Cards with WoW

Four metrics per page — Sessions, Top Funnel, Purchase Success, and CVR — each with WoW % change and directional color coding (green/red).

Dual-Axis Daily Chart

Chart.js bar + line combination: daily session volume on the left axis, daily CVR % on the right axis — 7-day trend per landing page.

Take-Rate Doughnut

Weekly product mix breakdown showing which subscription plans captured the most purchases — top 5 plans with percentage breakdown.

Category × Channel Table

Conversion performance by paid/organic category and acquisition channel — includes sessions, purchases, CVR, and WoW Δ delta columns.

Country Performance

Ranked country-level breakdown for multi-market tabs (LATAM, APAC, EMEA, Global) — showing sessions and CVR per market in one scan.

Automated Insight Boxes

Color-coded insight cards generated automatically: blue (neutral), green (wins), amber (monitor), red (alerts) — no manual copywriting needed.

Anomaly Detection

Z-score based statistical anomaly detection flags unusual CVR or session spikes/drops automatically, before they are caught in weekly review.
